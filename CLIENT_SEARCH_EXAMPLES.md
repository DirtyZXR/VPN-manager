# Примеры использования умного поиска клиентов

## Обзор
Умный поиск автоматически обрабатывает регистр, пробелы и спецсимволы, что делает поиск более удобным и прощающим ошибки пользователя.

## Примеры запросов

### Поиск по имени

**Базовый поиск:**
```
иван
```
Найдет: "Иван Петров", "Иванов", "Максим Иванин", "Ivan"

**Поиск по нескольким словам:**
```
иван петров
```
Найдет: "Иван Петров", "Петров Иван", "Петров Иван Сергеевич"

**С лишними пробелами:**
```
иван   петров
```
Автоматически нормализуется в: "иван петров"

**Со спецсимволами:**
```
иван,петров!
```
Найдет: "Иван Петров", "Петров Иван" (знаки препинания удалены)

**Смешанный регистр:**
```
ИВАН ПЕТРОВ
```
Найдет: "Иван Петров", "иван петров", "ИВАН ПЕТРОВ" (без учета регистра)

### Поиск по email

**Частичный email:**
```
test
```
Найдет: "test@example.com", "mytest@domain.com", "testing@vpn.com"

**Домен:**
```
@example.com
```
Найдет: все email с доменом "example.com"

**С лишними пробелами:**
```
  test@example.com
```
Автоматически нормализуется в: "test@example.com"

**В разном регистре:**
```
TEST@EXAMPLE.COM
```
Найдет: "test@example.com", "TEST@example.com", "test@EXAMPLE.COM" (приводится к lowercase)

### Поиск по Telegram ID

**Точное совпадение:**
```
123456789
```
Найдет: клиента с Telegram ID 123456789

**С пробелами (автоочистка):**
```
  123456789
```
Найдет: клиента с Telegram ID 123456789

**С буквами (ошибка):**
```
12345abc
```
Ошибка: "Telegram ID должен быть числом"

### Комплексный поиск

**Автоопределение типа:**
```
123456789
```
→ Поиск по Telegram ID (только цифры)

```
test@example.com
```
→ Поиск по email (содержит @)

```
иван петров
```
→ Поиск по имени (текст)

**Умный поиск по имени с несколькими словами:**
```
иван иванович петров
```
Найдет любого клиента, у которого в имени есть хоть одно из слов:
- "Иванович Иван Петров"
- "Петров Иван"
- "Иванин"
- "Иванов"

## Сценарии использования

### Сценарий 1: Не помните точное имя
**Проблема:** Нужно найти клиента, но вы помните только часть имени.

**Решение:**
```
иван
```
Найдет всех клиентов с "иван" в имени.

### Сценарий 2: Неверный регистр
**Проблема:** Клиент написал имя заглавными буквами.

**Решение:**
```
ИВАН ПЕТРОВ
```
Найдет "Иван Петров" (регистр игнорируется).

### Сценарий 3: Лишние пробелы
**Проблема:** Клиент случайно оставил лишние пробелы.

**Решение:**
```
иван    петров
```
Автоматически удалит лишние пробелы.

### Сценарий 4: Поиск по VPN email
**Проблема:** Нужно найти клиента по его VPN email из XUI.

**Решение:**
```
vpn-client@example.com
```
Найдет клиента, у которого есть VPN email, содержащий этот адрес.

### Сценарий 5: Поиск по домену email
**Проблема:** Нужно найти всех клиентов с email от определенного домена.

**Решение:**
```
@example.com
```
Найдет всех клиентов с email из домена example.com.

### Сценарий 6: Поиск по любому из слов
**Проблема:** Помните несколько слов из имени, но не уверены в порядке.

**Решение:**
```
петров иван сергеевич
```
Найдет любого клиента, у которого есть любое из этих слов в имени.

## Советы по поиску

✅ **Лучшие практики:**
- Используйте минимальную уникальную часть имени
- Для email можно использовать домен для группового поиска
- Для Telegram ID используйте только цифры
- Не бойтесь лишних пробелов и спецсимволов

❌ **Чего избегать:**
- Слишком короткие запросы (менее 2 символов)
- Поиск Telegram ID с буквами
- Слишком общие запросы (например, "а") - найдет слишком много результатов

## Технические детали

**Нормализация текста:**
- Удаление лишних пробелов
- Приведение к нижнему регистру (для email)
- Удаление знаков препинания (кроме @ и . для email)

**Многословный поиск:**
- Разбивает запрос на отдельные слова
- Ищет по любому из слов (OR логика)
- Также проверяет полное совпадение фразы

**Обработка ошибок:**
- Валидация Telegram ID на числовое значение
- Проверка на пустой запрос
- Понятные сообщения об ошибках
