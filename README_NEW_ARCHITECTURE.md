# Новая архитектура VPN Manager

## Обновленная архитектура

### Основные изменения:

1. **Client (VPN клиент)** - основная сущность вместо User
   - Имя, email, telegram_id
   - Может иметь несколько подписок
   - Управляется через меню "👤 Управление клиентами"

2. **Subscription (подписка)** - подписка клиента
   - Привязана к конкретному клиенту
   - Имеет уникальный токен
   - Может включать несколько inbound подключений
   - Настройки трафика и срока действия

3. **InboundConnection (подключение)** - уникальное подключение к inbound
   - Привязано к подписке и конкретному inbound
   - Создает клиента на 3x-ui панели
   - Уникальность: один inbound не может повторяться в одной подписке

4. **Inbound** - хранит информацию о клиентах
   - `client_count` - количество клиентов на inbound
   - Можно просматривать статистику клиентов

2. Можно добавлять, синхронизировать, тестировать серверы
3. После синхронизации видны все inbounds

## Примечания:

- **User** теперь используется только для админов/менеджеров
- **Client** - это VPN клиенты (основные пользователи системы)
- **Profile** и старые модели сохранены для обратной совместимости
- При добавлении inbound в подписку автоматически создается клиент на XUI панели
- Подписки хранятся в базе данных и синхронизируются с XUI панелями
