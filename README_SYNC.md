# Документация синхронизации данных

## Обзор

Система синхронизации обеспечивает консистентность данных между базой данных бота и панелями 3x-ui.

## Архитектура

### Стратегии синхронизации

- **Оптимистичные обновления (Optimistic Updates)**: Запись в БД, потом в XUI с rollback при ошибках
- **Read-Only Cache**: Сервера и inbounds как кэш из XUI
- **Двусторонняя синхронизация**: Подключения с проверкой новизны данных

### Статусы синхронизации

- `synced`: Данные синхронизированы
- `pending`: Синхронизация в процессе
- `error`: Ошибка при синхронизации
- `offline`: Сервер недоступен

## Использование

### Автоматическая синхронизация

```python
from app.services.sync_service import SyncService

sync_service = SyncService(session)
await sync_service.start_background_sync()
```

### Ручная синхронизация

```python
from app.services import SyncService

sync_service = SyncService(session)
results = await sync_service.manual_sync("server")
# results: {"synced": 5, "errors": 0, "details": []}
```

## Интеграция с ботом

### Команды синхронизации

- `🔄 Синхронизировать все` - Полная синхронизация
- `🖥️ Синхронизировать сервера` - Синхронизация только серверов
- `📊 Проверить целостность` - Проверка целостности данных

### Триггеры синхронизации

Синхронизация автоматически запускается:
- При изменении данных в боте
- Каждые 5 минут (фоновая задача)
- При запуске бота

## Мониторинг

### Логи

Синхронизация пишет подробные логи:
```
🔄 Начало цикла синхронизации в 2024-03-24 16:00:00
✅ Сервер 1 синхронизирован
📊 Статистика целостности: {'total': 10, 'synced': 8, 'error': 2}
✅ Цикл синхронизации завершен за 2.34s
```

### Статусы серверов

В админском меню можно видеть статусы:
- ✅ Синхронизирован
- ⚠️ Ошибка синхронизации
- 🔴 Недоступен

## Troubleshooting

### Сервера не синхронизируются

1. Проверьте статус сервера: `sync_status == "offline"`
2. Проверьте соединение с сервером
3. Посмотрите логи ошибок: `sync_error`

### Расхождения данных

1. Запустите полную синхронизацию
2. Проверьте целостность данных
3. При необходимости удалите проблемные записи
