# Комплексный поиск клиентов

## Обзор
Комплексный поиск ищет клиента **одновременно** по всем полям с логикой OR. Это значит, что если запрос совпадает с любым полем, клиент будет найден.

## Логика работы

### 1. Что проверяется
Комплексный поиск проверяет следующие поля одновременно:
- **Имя клиента** (с поддержкой нескольких слов)
- **Email клиента**
- **Telegram ID** (если запрос - число)
- **XUI email** (email из inbound подключений)

### 2. Логика поиска (OR)
Используется логика OR - клиент найден, если совпадает **хотя бы одно** поле:

```sql
WHERE
    LOWER(name) LIKE LOWER('%query%')
    OR
    LOWER(email) LIKE LOWER('%query%')
    OR
    telegram_id = query
    OR
    LOWER(inbound_connections.email) LIKE LOWER('%query%')
```

## Особенности

### Многословный поиск по имени
При вводе нескольких слов поиск работает следующим образом:
```
Ввод: "денис петров"

SQL эквивалент:
WHERE LOWER(name) LIKE LOWER('%денис%')
   OR LOWER(name) LIKE LOWER('%петров%')
   OR LOWER(name) LIKE LOWER('%денис петров%')
```

То есть ищет:
- Любое слово отдельно
- И полную фразу

### Telegram ID особенность
Если запрос - только цифры, то дополнительно ищется по Telegram ID:
```
Ввод: "123456"

Результаты:
✅ Найдет по имени, содержащему "123456"
✅ Найдет по email, содержащему "123456"
✅ Найдет по XUI email, содержащему "123456"
✅ Найдет по точному Telegram ID "123456"
```

### Нормализация текста
Все запросы проходят через нормализацию:
- Удаляются лишние пробелы
- Удаляются спецсимволы (кроме @ и . для email)
- Слова короче 2 символов игнорируются (для многословного поиска)

### JOIN для XUI email
Поиск по XUI email требует JOIN с таблицей `inbound_connections`:
```sql
SELECT DISTINCT clients.*
FROM clients
JOIN subscriptions ON clients.id = subscriptions.client_id
JOIN inbound_connections ON subscriptions.id = inbound_connections.subscription_id
WHERE LOWER(inbound_connections.email) LIKE LOWER('%query%')
```

## Преимущества комплексного поиска

1. **Максимальный охват**: Находит клиентов по любому доступному полю
2. **Интуитивность**: Пользователю не нужно думать, по какому полю искать
3. **Гибкость**: Работает с частичными совпадениями
4. **Многословность**: Поддерживает поиск по нескольким словам в имени
5. **Универсальность**: Одна строка поиска для всех типов данных

## Когда использовать комплексный поиск

✅ **Идеально для:**
- Когда не уверены, по какому полю искать
- Когда нужно найти клиента по минимальной информации
- Когда нужна максимальная гибкость поиска
- Когда клиенты могут иметь похожие данные в разных полях

❌ **Не рекомендуется для:**
- Точного поиска по Telegram ID (лучше использовать отдельный поиск)
- Поиска конкретного email (лучше использовать отдельный поиск)
- Когда нужно сузить результаты (лучше использовать поиск по конкретному полю)

## Техническая реализация

Метод: `ClientService.search_clients_all_fields(query: str)`

Параметры:
- `query`: Строка поиска

Возвращает: `Sequence[Client]` - список найденных клиентов

Использует SQLAlchemy с текстовыми запросами для надежной работы с кириллицей:
```python
text("LOWER(clients.name) LIKE LOWER(:name)").params(name=f"%{query}%")
```
